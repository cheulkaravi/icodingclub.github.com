Introduction


Pass-by-Value vs Pass-by-refrence: Javascript and Java

Later or sooner, you will be asked, "Hey, Is this langauge is pass-by-value or pass-by-refrence".

Even after years, people reply something like, "It's pass by value......or wait a minute...it's pass by refrence....Hey you know what, i read it some xyz blog, it's always be pass by value".

This article will help you to understand that./p>

How it work


So what is it?....pass by value or pass by refrence?

In Javascript and java, we have two types, premitive type and refrence/Object type

premitive type is easy, it allways be pass by value, simple and clear. Hemce we are not going to discuss that

What about refrence?

For all theoritcal reason(interview and test), just say pass by value

However, for all practical resaon, you have to go bit deeper, this article is all about that


Belive me, above statement is not important, what important is to understand How this all work!

Let's understand first, What actually happen when we create Object and refrence.

Let's go with example where you have a TV in your home, its far and not many button on it. To access it, you need a remote.

A refrence of objcet is nothing but high-tech, re-configurable remote, which can point to your TV, but when re-configured, it can point to another TV, but at a time only one TV and TV is nothing but the Object, sitting deep down in Heap.




Step 1: You create a TV(by calling new Object()), you configured this TV with remote1(by assigning Object to refrence)

At this stage, you can change the volume of TV by using remote (You can change the value of Object by refrence)


Step 2: Now You want a separete remote, you create a copy of it, at this time both remote has power to change the volume of TV, mind it none of the remote is a TV, breaking a remote will not affect TV(as assigning refrence to null, will not change the object or assigned it to null)

Step 3: because of some reason, you configured second remote to second TV (assigning remote2 refrence to new TV Object)

At this stage relation of both remote are gone, Each can control there own TV but not of each others. if remote2 get destroyed(remote2 = null) and then TV2 get destroyed(2nd TV object get destroyed), remote1 and TV1 will stil work fine.

Code walkthrough


Everything we discussed above, can be connected in code

For simplicity, i have taken similar use case of TV, with a vol property.

Both in Javascript and Java code, we are passing refrence to other function/method and try to change the vol property.




Javascript



/**
 * A simple Object/function, we call it TV, with propert vol
 */
function TV(vol) {
    this.vol = vol;
}

var remote1 = new TV(10);
console.log("Initial test: ", remote1.vol);                 // Print 10
duplicate(remote1);
console.log("After Duplicate: ", remote1.vol);              //Print 20

/**
 * This is pass by value, but work like pass-by-refrence i.e. whatever changes you will do here to remote2 will affet the main TV
 * i.e. remote1 will also reflect same.
 */
function duplicate(remote2) {
    remote2.vol = 20;
    console.log("In Duplicate: ", remote2.vol);             //Print 20
}

See what happen when we assign another object to refrence, it's lost all connection with initial Object

Point to note, first refrence(remote1) is stil pointing to first Object and it's unchanged value

History of this debate, Let us 'C'


Any one from C background, will ask you this one day

Why??...bacuse C started pass-by-val and pass-by-ref concept and It does it well

It is clear in code when you are doing what and why you doing so..

i tried to have similar code what we have seen so far, struct is closet o Object in C



#include <stdio.h>
#include <stdlib.h>

struct TV {
  int vol;
};
 
void duplicate(struct TV remote2) {                 //Pass by value, crystall clear
    remote2.vol = 20;
    printf("In Duplicate: %d\n", remote2.vol);      //Print 20
}
 
int main()
{
    struct TV remote1;
    remote1.vol = 10;  
    printf("Initial test: %d\n", remote1.vol);      //Print 10
    duplicate(remote1);
    printf("After Duplicate: %d\n", remote1.vol);   //Print 10
}

     

#include <stdio.h>
#include <stdlib.h>

struct TV {
  int vol;
};
 
void duplicate(struct TV *remote2) {                //Pass by ref, crystall clear
    remote2->vol = 20;
    printf("In Duplicate: %d\n", remote2->vol);     //Print 20
}
 
int main()
{
    struct TV remote1;
    remote1.vol = 10;  
    printf("Initial test: %d\n", remote1.vol);      //Print 10
    duplicate(&remote1);                            // passing address
    printf("After Duplicate: %d\n", remote1.vol);   //Print 20
}
